# Task Instructions

## Goal
Create a semantic chat for querying legal verdicts

## Requirements

### 1. Document Download
- Download 50 Word documents and 50 PDF documents to a local folder
- Use the scraper script located in this directory
- The files must be unique (not the same file downloaded multiple times)

### 2. Semantic Database
- Create a semantic database for the files
- Enable querying of the files
- Purpose: to ask questions about the verdicts

### 3. Efficiency and Execution
- The solution must be efficient
- Must be able to run locally on the computer
- If needed, also in a cloud environment

### 4. Model Abstraction
- There must be an abstraction layer to easily switch between models

### 5. User Interface
- Create a chat user interface for querying the files
- The user interface must be clean and minimal
- Must not contain any bugs

### 6. Streaming
- Enable streaming of model responses in the user interface
- The chat responses must reference their sources

### 7. Local Execution
- **Mandatory** that the system can run end-to-end locally

### 8. Documentation
- **Mandatory** to include basic execution instructions in the repo

### 9. Code Quality
- The code **must** be clean and organized

### 10. Repository
- The commit must include everything, including the downloaded files
